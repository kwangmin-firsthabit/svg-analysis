## 작업 지시

인자로 받은 시스템 프롬프트와 유저 프롬프트를 참고하여 SVG 파일을 생성한다.

### 인자 파싱

`$ARGUMENTS`에서:
- 첫 번째 인자: 카테고리명 (예: `삼각형`, `사각형`, `원`)
- 두 번째 인자: 저장 경로 (예: `data/static`, `data/01.static`)
- 세 번째 인자: 시스템 프롬프트 파일 경로 (예: `docs/prompts/system/prompt.static.md`)
- 네 번째 인자: 유저 프롬프트 파일 경로 (예: `docs/prompts/user/prompt.user.md`)

### 실행 절차

1. **프롬프트 파일 읽기**
   - 세 번째 인자 경로의 시스템 프롬프트 파일을 읽는다.
   - 네 번째 인자 경로의 유저 프롬프트 파일을 읽는다.

2. **카테고리 항목 추출**
   - 유저 프롬프트에서 첫 번째 인자로 지정된 카테고리 섹션의 항목 목록을 파악한다.

3. **출력 디렉토리 및 기존 파일 확인**
   - 두 번째 인자로 받은 경로의 파일 목록을 확인한다.
   - 이미 존재하는 파일은 건너뛴다.

4. **파일명 결정**
   - 형식: `NN.이름.html` (예: `01.정삼각형.html`, `02.직각삼각형.html`)
   - 번호는 카테고리 내 순서 기준 01부터 시작한다.
   - 이름은 항목의 핵심 명칭을 간결하게 사용한다 (공백 없이).

5. **SVG 생성 규칙** (시스템 프롬프트 원칙 준수)
   - **SVG 요소만 출력한다. `<!DOCTYPE>`, `<html>`, `<head>`, `<body>` 태그를 사용하지 않는다.**
   - 파일 최상단에 좌표 계산 검증 주석을 포함한다.
   - 수학적 요소(도형, 레이블, 기호)만 포함한다. 설명 텍스트 금지.
   - css 변수 사용 금지.
   - 기하 관계(외접, 내접, 수직 등)는 방정식으로 좌표를 도출하고 수치로 검증한다.
   - 텍스트는 꼭짓점 레이블, 변수명, 수치값, 축 레이블만 허용한다.

6. **파일 저장**
   - 각 항목을 개별 파일로 저장한다.
   - 저장 후 생성된 파일 목록을 보고한다.
